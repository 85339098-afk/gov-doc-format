# GB/T 33476.2-2016 关键条款摘录

> **党政机关电子公文格式规范 第2部分：显现**
>
> 本标准规定了使用文字处理软件制作显示效果符合 GB/T 9704-2012 的电子公文时应遵循的原则、
> 推荐的参数以及公文域实现要求。

---

## 1. 基本信息

- 标准号：GB/T 33476.2-2016
- 发布日期：2016-12-30
- 实施日期：2017-07-01
- 归口单位：国家电子文件管理部际联席会议办公室
- 起草单位：中办秘书局、国办秘书局、中办信息中心等

## 2. 与本 skill 高度相关的条款

### 2.1 页面设置参数（第6章）

| 项目 | 推荐参数 |
|------|---------|
| 版心 | 156mm × 225mm |
| 上边距 | 37mm |
| 下边距 | 35mm |
| 左边距 | 28mm |
| 右边距 | 26mm |
| 每页行数 | 22 行（3 号仿宋字） |

### 2.2 字体设置（第7章）

与 GB/T 9704-2012 一致，提供文字处理软件中的实现建议。

### 2.3 行距计算（附录A）

GB/T 9704-2012 中的"一行"定义为：一个 3 号字高 + 3 号字字高的 7/8。

| 参数 | 数值 |
|------|------|
| 3 号字字高 | ≈ 5.54mm |
| 3 号字字高的 7/8 | ≈ 4.84mm |
| 一行总高度 | ≈ **10.39mm** |
| Word 固定值换算 | ≈ **29.5pt** |

### 2.4 正文上边距（附录A）

文字处理软件中，第一行文字顶边与上边距下边缘之间有 3 号字字高的 7/16 的距离（≈ 2.42mm）。

### 2.5 版心高度验证

版心高度 = 22 行 × 10.39mm = 228.58mm。实际版心标准为 225mm，
差异来自首行上方的 7/16 字高间距和末行下方的 7/16 字高间距。

## 3. 公文域要求（第10章）

定义了 25 个标准公文域的命名规则，包括：

| 序号 | 公文要素 | 公文域标识 |
|------|---------|-----------|
| 1 | 份号 | fenhao |
| 5 | 发文机关名称 | fawenjiguanmingcheng_n |
| 11 | 标题 | biaoti |
| 13 | 正文 | zhengwen |
| 15 | 发文机关署名 | fawenjiguanshuming_n |
| 17 | 成文日期 | chengwenriqi |
| 23 | 印发机关 | yinfajiguan |
| 24 | 印发日期 | yinfariqi |

> 注意：公文域功能依赖文字处理软件的支持，本 skill 的 Python 脚本暂未实现公文域功能。

## 4. 与本 skill 的关系

- GB/T 33476.2-2016 是 GB/T 9704-2012 的配套标准，**不是替代关系**
- 本标准解释了为什么在 Word 中设置固定值 28pt 或 29.5pt 都能达到近似效果
- 本标准提供了文字处理软件中实现公文格式的权威参考
